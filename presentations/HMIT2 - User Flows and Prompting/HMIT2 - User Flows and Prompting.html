What Is an LLM?

  • An LLM (large language model) predicts the next token in a sequence
  • A token can be a short word or part of a word
  • It generates text

Example: If the input is "A login screen usually has email, password, and a..." a likely next prediction could be "button".

How Does the Model Know Things?

  • The model was trained on very large amounts of text, code, and other data
  • During training it learned patterns, relationships, and common structures
  • It learns statistical patterns that let it generate plausible answers
Large Language Model ≠ Large Knowledge Model

How AI Chat Tools Work

  • Chat is still next-token prediction in a special format
  • The system adds user: and assistant: to the messages before the model sees them

Example: When asking an AI chat to “Create a recipe for Pizza Margherita”, the system modifies the message so it looks like

user: Create a recipe for Pizza Margherita
assistant:

The model is not "talking" like a person. It is predicting the most likely assistant response in that format.

Without the formatting an LLM would simply try to continue the text. Create a recipe for Pizza Margherita that is simple, quick, and suitable for beginners.

Important Insight

  • The chat system re-sends the conversation history each time
  • Longer prompts and longer chats cost more and give the model more to process
  • This is why old context still matters and why messy chats get worse over time
  • Contradictory instructions weaken the result
  • Starting a fresh chat is often better than repairing a bad one forever

System Prompts

  • Every AI tool also has hidden instructions
  • These system prompts tell the model how to behave
  • They influence tone, structure, and preferred output
  • Your prompt works together with those hidden rules, not in isolation

Example: A complete message reaching the model could look like this:

system: You are a helpful assistant. Keep answers short and structured. Do not help with harmful, illegal, abusive, or dangerous requests.
user: Create a recipe for Pizza Margherita
assistant:

AI Chat Tools Today

Leading AI chat tools like ChatGPT, Gemini, or Claude are more than autocomplete systems. They are built as a combination of tools. The chat is the interface people interact with, while the system behind it may decide when to search, reason, generate code, inspect files, or work with images.

AI Chats Tools Today

Image Generation

  • In AI chat tools, image recognition or image editing is handled by a different model, not by the language model
  • The LLM interprets the user's instruction and coordinates the task
  • A vision or image model (CLIP, SigLIP) analyzes or edits the image
  • The result is then returned to the chat interface with a language explanation if needed
flowchart LR
    A["User: Remove the person in the background"] --> B["Chat interface"]
    B --> C["LLM interprets the request"]
    C --> D["Image model detects objects and edits image"]
    D --> E["Edited image returned"]
    E --> F["LLM explains result or asks for refinement"]

Skills/Tools

  • LLMs can work with other applications through tools
  • These tools may be called MCP, ACP, function calling, actions, or integrations
  • The LLM does not directly operate the app itself, instead it generates an instruction and the tool system executes it

Example: Working with Figma MCP

  • The MCP acts as a translator between the LLM and Figma's API
  • The LLM does not draw directly, it generates tool instructions
  • The MCP server executes those instructions as Figma node changes
  • The loop is: read canvas state -> reason over structured data -> write updates -> check result
  1. Prompt: Draw a blue rectangle
  2. LLM searches for siutable Figma tools
  3. LLM “responds” with a tool call:
{
  "tool": "create_rect",
  "arguments": {
    "width": 200,
    "height": 100,
    "x": 0,
    "y": 0,
    "fills": [{"type": "SOLID", "color": {"r": 0, "g": 0, "b": 1}}]
  }
}
  1. System around the LLM catches the “response” and hands it over to the Figma MCP server
  2. Figma executes the task

Prompting

  • You are not programming the model, you are directing it
  • Better direction leads to better results
  • Your job is to provide context, constraints, and judgment

Prompting

For UX Prototypes

  • Describe the goal
  • Describe the user
  • Describe the problem
  • Describe the main flow
  • Describe the visual tone
  • Describe the constraints of the MVP

Prompting

For UX Prototypes

A Weak Prompt

Create an app for roommates

  • Unclear user need
  • No main task
  • No interface direction
  • No scope

Prompting

For UX Prototypes

A Better Prompt

Create a simple web app prototype for roommates who want to track shared groceries. The MVP should let users add an item, mark who bought it, and see who owes what. Use a clean mobile-first layout with a list view, a summary area, and only the essential actions.

Prompt Structure

  1. Describe the goal
  2. Describe the context
  3. Describe the constraints
  4. Describe the desired output
  5. Add examples if they help

Prompting

Best Practices

  • Understand the model’s capabilities
  • Stay within one clear scope
  • Ask for one thing at a time
  • Include as much context as possible
  • Include constraints
